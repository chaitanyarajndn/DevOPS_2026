



==================== TERRAFORM_STATE_RELATION_NOTES ====================

Purpose:
These notes explain the relationship between:

* .tf file        → Desired State (what you define)
* .tfstate file   → Known State (what Terraform tracks)
* Azure Portal    → Actual Infrastructure (real resources)

Terraform always tries to make:
→ Desired State (.tf) == Actual Infrastructure (Azure)

---

## CORE CONCEPT: EQUILIBRIUM IN TERRAFORM

Technical Term:
→ "State Reconciliation" (also commonly referred to as "Desired State Convergence")

Meaning:
Terraform continuously works to bring all three components into alignment:

1. Desired State (.tf)
2. Known State (.tfstate)
3. Actual Infrastructure (Azure)

When all three match:
→ System is in "Equilibrium" (Fully Converged State)

When they don’t match:
→ Terraform detects "Drift" and performs reconciliation

Flow:

* Refresh → Syncs .tfstate with Azure (real-world infra)
* Plan    → Compares .tf (desired) vs .tfstate (current)
* Apply   → Takes actions to achieve convergence

Key Understanding:
→ Terraform is a Declarative Tool
→ You declare WHAT you want, Terraform ensures it EXISTS

---

SCENARIO 1:
Resource present in .tf file BUT NOT in .tfstate and NOT in Azure Portal
------------------------------------------------------------------------

Situation:

* Resource is defined in configuration (.tf)
* Resource does not exist in state file
* Resource does not exist in Azure

(This is the NORMAL case when creating a new resource)

What Happens:

* `terraform plan` → Marks resource as "to be created"
* `terraform apply` → Creates resource in Azure
* Terraform updates .tfstate with new resource entry

Professional Explanation:
Terraform detects that the resource is part of the desired state but missing from both the current state and actual infrastructure. It provisions the resource and records it in the state file, achieving convergence.

Key Point:
→ Defined in .tf only = Terraform will CREATE it

Precautions:

* Ensure resource naming is unique (avoid Azure conflicts)
* Validate configuration (`terraform validate`)
* Always review `terraform plan` before apply

Post-Cautions:

* Verify resource in Azure Portal
* Ensure remote state is updated correctly
* Avoid immediate manual modifications

---

SCENARIO 2:
Resource present in .tfstate and .tf file BUT NOT in Azure Portal
-----------------------------------------------------------------

Situation:

* Resource exists in Terraform state
* Resource exists in configuration
* Resource is missing in Azure (manual deletion / drift)

What Happens:

* `terraform refresh` → Detects missing resource
* `terraform plan` → Marks resource for recreation
* `terraform apply` → Recreates resource in Azure

Professional Explanation:
This is a case of "Infrastructure Drift." Terraform identifies inconsistency between desired and actual state and reconciles it by recreating the missing resource, restoring equilibrium.

Key Point:
→ Missing in Azure but defined = Terraform will RE-CREATE it

Precautions:

* Avoid manual deletion in Azure Portal
* Enforce IaC-only changes
* Use remote backend with state locking

Post-Cautions:

* Check dependency impacts (IDs, IPs may change)
* Validate integrations (NSG, IAM, etc.)
* Review plan output carefully

---

SCENARIO 3:
Resource present in Azure Portal BUT NOT in .tf and NOT in .tfstate
-------------------------------------------------------------------

Situation:

* Resource exists in Azure
* Resource is NOT defined in Terraform
* Resource is NOT tracked in state

(This is an unmanaged resource)

What Happens:

* Terraform does NOTHING (completely ignored)
* No changes in `plan` or `apply`

Professional Explanation:
Terraform only manages resources declared in configuration and tracked in state. Any external resource is outside its control unless explicitly imported. Hence, no reconciliation occurs.

Key Point:
→ Not in .tf and not in state = Terraform IGNORES it

Precautions:

* Avoid manual resource creation outside Terraform
* Maintain Infrastructure as Code discipline
* Regularly audit Azure environment

Post-Cautions:

* Use `terraform import` if resource must be managed
* Prevent duplicate creation via Terraform later
* Document unmanaged resources clearly

## FINAL UNDERSTANDING (BIG PICTURE)

```
           Desired State        Known State        Actual Infra
               (.tf)            (.tfstate)          (Azure)
                  \                 |                 /
                   \                |                /
                    \_________ RECONCILIATION ______/
```

Terraform Goal:
→ Bring system to "Desired State Convergence" (Equilibrium)

Key Terms:

* Drift                → Mismatch between Azure & State
* Refresh             → Sync state with real infra
* Plan                → Compare desired vs current
* Apply               → Enforce desired state
* Convergence         → Final aligned state (Equilibrium)

Golden Rule:
→ If it's in .tf → Terraform will enforce it
→ If it's not → Terraform ignores it
