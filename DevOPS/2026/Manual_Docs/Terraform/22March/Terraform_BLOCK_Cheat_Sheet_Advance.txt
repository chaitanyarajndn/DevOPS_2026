==============================================================
[ 1) SINGLE-LABEL BLOCK : terraform {} ]
========================================

> > > PURPOSE

---

✔ Global configuration (NO labels)
✔ Defines provider + backend
✔ DOES NOT create infra  ❗
✔ Executed ONLY during terraform init

✔ Controls state storage + locking behavior
✔ Ensures consistent provider + Terraform versions
✔ Mandatory for team-based environments

> > > BACKEND INTERNAL (DEEP)

---

✔ tfstate stored in Azure Blob (single source of truth)
✔ Uses Blob Lease ➜ exclusive lock mechanism
✔ Prevents concurrent apply operations
✔ Ensures atomic updates to state file
✔ Critical for avoiding infra drift + corruption

> > > CODE (INLINE WHAT + WHY)

---

terraform {

required_version = ">= 1.5.0"                 # Enforces minimum Terraform version ➜ prevents incompatible state operations

required_providers {
azurerm = {
source  = "hashicorp/azurerm"             # Official provider source ➜ ensures trusted plugin binary
version = "~> 3.0"                        # Locks major version ➜ avoids breaking API/schema changes
}
}

backend "azurerm" {


resource_group_name  = "rg-tfstate-demo"    # MUST exist BEFORE init ➜ backend cannot create infra resources

storage_account_name = "sttfstatedemo123"   # Globally unique ➜ Azure enforces DNS-level uniqueness

container_name       = "tfstate"            # Logical container ➜ stores multiple tfstate files (env separation)

key                  = "dev.terraform.tfstate" # Unique state file ➜ maps to specific environment (dev/prod)


}
}

---
